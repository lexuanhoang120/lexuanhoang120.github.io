Web Face Recognition

VTCODE Company · 2022.10

A full-stack web application for face recognition, built with a FastAPI backend, a browser-based frontend, and MongoDB for storing user and recognition data. The project connects face recognition inference with practical web workflows such as login, user listing, profile information, and database management.

Role note: I was responsible for the backend side (API and data workflow). The frontend interface was developed by other team members. Project status: Pending.

GitHub Demo Approach Results

Login

Users start from the login interface before accessing the recognition dashboard.

User List

The list page displays registered or recognized users and supports browsing user records through the web interface.

User Information

The information page allows users to view or add profile details connected to face recognition records.

Database

MongoDB is used to store user profiles, recognition records, and related system data.

Main Features

  • Login: authentication interface for accessing the system.
  • List: view registered or recognized users.
  • Information: add and manage user profile details.
  • Database: store user and recognition data using MongoDB.

Approach

The project separates the backend and frontend so recognition logic, database operations, and user interaction can be managed cleanly.

System Design

  • Backend: FastAPI service for recognition-related API logic.
  • Frontend: web interface for login, list, information.
  • Database: MongoDB for storing user and recognition records.
  • Workflow: user action → API request → data processing → database update → frontend display.

Results

The project produced a working web prototype that integrates face recognition workflows with a browser interface and database-backed user management.

Key Takeaways

  • Built a full-stack face recognition web application.
  • Implemented a FastAPI backend for web-based recognition workflows.
  • Used MongoDB to manage user and recognition-related records.
  • Created demo flows for login, user list, information, and adding new information.
