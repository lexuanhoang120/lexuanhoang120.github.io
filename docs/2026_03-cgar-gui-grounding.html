CGAR: Confidence-Guided Adaptive Refinement for GUI Grounding

Academic Manuscript · 2026.03 · CVPR 2026 Submission

CGAR is a training-free refinement method for GUI grounding in computer-use agents. It improves click-point prediction by using model confidence to decide whether a prediction should be accepted directly or refined through an adaptive local crop.

GitHub Approach Results

Approach

GUI agents often fail when they click the wrong UI element, especially on dense and high-resolution screens. Traditional reflection methods try to fix mistakes after execution, but this can be expensive and may already change the interface state. CGAR instead improves the prediction before execution.

Core Idea

CGAR uses the confidence score from an attention-based GUI grounding model as a control signal. If the model is confident, the prediction is finalized. If confidence is low, CGAR crops around the predicted area and asks the model to predict again on the clearer local view.

High-Level Workflow

  1. Input a GUI screenshot and natural-language instruction.
  2. Predict an initial target point from the full screenshot.
  3. Estimate confidence from the attention map around the prediction.
  4. Stop early if the confidence is high enough.
  5. Otherwise, crop around the predicted point and refine the prediction.
  6. Repeat until confidence is high or the maximum refinement depth is reached.

What Makes It Useful

  • Training-free: improves inference without retraining the base model.
  • Confidence-guided: refines only uncertain predictions instead of applying fixed refinement to every case.
  • Adaptive cropping: uses larger crops for uncertain cases and tighter crops when the model becomes more confident.
  • Better dense-screen grounding: helps localize small or visually similar UI elements in complex layouts.

Evaluation Scope

  • Models: GUI-Actor and V2P.
  • Benchmarks: ScreenSpot-V2, ScreenSpot-Pro, and UI-Vision.
  • Metric: grounding accuracy, based on whether the predicted point falls inside the target bounding box.

Results

CGAR delivered the strongest improvements on dense, high-resolution GUI benchmarks. The gain was smaller on ScreenSpot-V2 because the baseline performance was already high.

Main Results

Model Dataset Baseline CGAR Gain
GUI-Actor-7B ScreenSpot-Pro 44.6% 60.5% +15.9%
V2P-7B ScreenSpot-Pro 50.5% 64.2% +13.7%
GUI-Actor-7B UI-Vision 21.4% 31.0% +9.6%

Key Takeaways

  • CGAR significantly improved GUI grounding without additional model training.
  • V2P-7B + CGAR reached 64.2% on ScreenSpot-Pro, outperforming several larger models reported in the paper.
  • GUI-Actor-3B + CGAR outperformed the baseline GUI-Actor-7B on ScreenSpot-Pro and UI-Vision.
  • Compared with fixed-scale refinement, CGAR was more stable because it adapts the crop size based on confidence.
  • The method is most useful for dense professional interfaces where the first full-screen prediction is often uncertain.

Limitations

CGAR depends on the initial prediction as the starting point, so very poor first predictions can still be difficult to recover from. It also adds extra inference steps, although confidence-based stopping helps reduce unnecessary refinement.
