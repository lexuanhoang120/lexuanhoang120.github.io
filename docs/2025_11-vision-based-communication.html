Vision-based Communication with Object Tracking

Sejong University VLI Lab · 2025.11

This project combines computer vision and wireless communication to improve beam alignment for moving vehicles. Instead of steering the beam using delayed detections, the system predicts where the vehicle will be at beamforming time, then points the beam to that estimated position. The result is a clear increase in average data rate.

GitHub Problem Approach Results

Problem

Beamforming works best when the antenna points exactly at the target. In practice, targets move, and there is always a short delay between perception and action. That delay causes pointing errors, reduces antenna gain, and lowers the final communication rate.

Project Objective

  • Reduce beam misalignment caused by motion and processing latency.
  • Predict a vehicle's near-future location before assigning the beam.
  • Compare the predictive method against a delayed-detection baseline.
  • Measure communication performance on real driving scenarios.

Approach

The pipeline starts from a camera stream and ends with a communication-rate estimate. Vehicles are detected, tracked across frames, projected into directional space, and evaluated with a beamforming model. The key step is future-location estimation, which compensates for control delay.

High-Level Pipeline

  1. Detect vehicles in camera frames using an object detection model.
  2. Track detected vehicles across frames to preserve object identity over time.
  3. Estimate the future bounding box of each tracked vehicle using recent motion history.
  4. Convert 2D image positions into 3D directional information.
  5. Calculate azimuth and elevation angle differences between predicted and ground-truth target positions.
  6. Estimate antenna gain based on angular misalignment.
  7. Compute average data rate across a range of SNR values.

Computer Vision Components

  • Object detection: YOLO11 detects vehicles in each frame (focused on the car class).
  • Object tracking: ByteTrack keeps identity continuity across frames.
  • Future location estimation: a linear predictor estimates the next bounding box from recent motion.

Baseline vs Proposed Scheme

  • Baseline: uses the last detected location, so beam direction can lag behind moving targets.
  • Proposed method: predicts the vehicle position at action time and steers the beam there.

Evaluation Dataset

Experiments were run on the DeepSense 6G dataset, which includes synchronized camera, GPS, LiDAR, radar, and wireless communication signals. We used three real-world scenarios at 960x540 resolution, each with thousands of frames.

Evaluation Setup

  • Scenario A: DeepSense 6G Scenario 13.
  • Scenario B: DeepSense 6G Scenario 9.
  • Scenario C: DeepSense 6G Scenario 3.
  • Tracking interval: 1/30 second.
  • Rate calculation interval: 10/30 second.
  • SNR range: 0, 5, 10, 15, 20, 25, and 30 dB.
  • Metric: average data rate in bps/Hz.

Results

The proposed method outperformed the delayed-detection baseline in all three scenarios. Gains were consistently above 20%, showing that short-horizon motion prediction is effective for reducing beamforming error.

Overall Performance

Scenario Baseline Rate Proposed Rate Improvement
Scenario A 2.786 bps/Hz 3.592 bps/Hz +28.93%
Scenario B 2.999 bps/Hz 3.605 bps/Hz +20.19%
Scenario C 2.759 bps/Hz 3.529 bps/Hz +27.93%

Key Findings

  • All scenarios show higher throughput with prediction-based steering.
  • Best gain: Scenario A with 28.93% improvement.
  • Most conservative gain: Scenario B with 20.19%, still substantial.
  • Scenario C remains strong at 27.93%, close to Scenario A.
  • Future-position estimation is consistently better than relying on delayed detections.

Conclusion

This project shows that adding lightweight prediction to a vision pipeline can materially improve communication quality. By aligning beam steering with where the vehicle will be, not where it was, the system reduces pointing error and raises average data rate by 20%+ across diverse real-world conditions.
