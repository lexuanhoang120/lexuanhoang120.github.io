VLM-Based CCTV Video Filtering

PIA Space · 2025.12 - 2026.02

A VLM-based pipeline for building CCTV anomaly datasets from web videos. The system searches for candidate videos, filters surveillance-style footage, and extracts short anomaly clips for classes such as fire, smoke, violence, and falldown.

This project was carried out as an internship task at PIA Space, assigned through the Vision Language Intelligence Laboratory.

GitHub Demo Approach Results

Demo

The demo shows how a user query is converted into a cleaner CCTV anomaly dataset: download candidate videos, keep only CCTV footage, then extract short clips that contain the target event.

High-Level Flow

  1. Input an anomaly class such as fire, smoke, violence, or falldown.
  2. Generate search queries and download candidate videos.
  3. Sample representative frames from each video.
  4. Use VLM prompts to classify CCTV vs. non-CCTV footage.
  5. Run anomaly detection on filtered CCTV videos.
  6. Save short event clips into class-specific folders.

Goal

  • Find useful CCTV videos from noisy web search results.
  • Reduce manual annotation cost for rare anomaly events.
  • Create 1-5 second anomaly clips with visual evidence.
  • Target 20-30 useful videos per class.

Approach

The project is organized into three modules: downloader, filtering, and refinement. Together, they form an end-to-end workflow for CCTV anomaly dataset construction.

Downloader

Searches the web for videos related to the target anomaly. LLM-generated queries improve coverage by using CCTV-specific terms such as surveillance camera, security footage, fixed camera, and incident context.

Filtering

Removes non-CCTV videos before downstream processing. The module samples frames from each video and uses VLM-based VQA prompts to decide whether the video is surveillance footage. A grayscale-difference heuristic was also tested as a lightweight baseline.

Refinement

Splits full CCTV videos into short anomaly clips. The module samples 64 frames, asks a VLM to identify frames containing Fire, Smoke, Violence, or Falldown, maps detected frame IDs to timestamps, and cuts clips around the detected moments.

Results

The filtering module was tested with InternVL3 and Qwen3-VL models across multiple prompts. Prompt design had a strong impact on accuracy.

Key Takeaways

  • Built a three-stage pipeline for web video search, CCTV filtering, and anomaly clip extraction.
  • Qwen3-VL-2B achieved 94.44% accuracy on the small CCTV filtering test set.
  • InternVL3-2B achieved 83.53% accuracy on the larger CCTV-only dataset.
  • Prompt design strongly affected CCTV filtering performance across models.
