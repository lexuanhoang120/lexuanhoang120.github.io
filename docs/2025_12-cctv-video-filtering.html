VLM-Based CCTV Video Filtering

PIA Space (Lab Internship) · 2025.12 – 2026.02

A VLM-powered pipeline that automatically classifies videos as CCTV or non-CCTV footage. Uses Qwen3-VL-2B and InternVL3-2B with frame sampling and prompt-based inference. Achieved 94.44% accuracy on a curated test set and 83.53% on a larger 425-video CCTV-only dataset — prompt engineering with surveillance-specific terminology was the key driver of accuracy.

  • Prompt engineering had a strong impact — optimized prompts incorporating CCTV terminology (surveillance camera, security footage, fixed camera, incident context) significantly outperformed generic ones.
  • A grayscale-difference heuristic was tested as a lightweight non-VLM baseline — useful for pre-filtering but the VLM approach delivered higher accuracy.
  • Target anomaly classes: Fire, Smoke, Violence, Falldown.

How it works

  1. Input an anomaly class (fire, smoke, violence, or falldown)
  2. Generate search queries with CCTV terminology and download candidate videos
  3. Sample representative frames from each video
  4. Use VLM prompts to classify CCTV vs. non-CCTV footage
  5. Run anomaly detection on filtered CCTV videos (64 frames per video)
  6. Save 1–5 second event clips into class-specific folders
