DART for GUI Grounding

Vision Language Intelligence Lab · 2026.05 · First-author research

DART is a training-free refinement method for GUI grounding in computer-use agents. It improves click-point prediction by using attention density as a confidence signal, then adaptively refining uncertain regions before the agent takes action.

GitHub Demo Approach Results

Demo

Given a screenshot and an instruction, DART first predicts a target UI location. If the model is uncertain, it crops around the predicted area, re-runs grounding on the clearer local view, and repeats until the prediction becomes confident enough.

High-Level Flow

  1. Input a GUI screenshot and natural-language instruction.
  2. Predict an initial target point from the full screenshot.
  3. Measure confidence from the attention density around the prediction.
  4. Stop early if confidence is high.
  5. Otherwise, crop around the target and refine the prediction.
  6. Map the final local prediction back to the original screenshot.

Why It Matters

  • More reliable agents: reduces wrong clicks before execution.
  • Better dense-GUI grounding: helps with small and crowded UI elements.
  • No extra training: works as an inference-time refinement strategy.
  • Adaptive behavior: uses wider crops when uncertain and tighter crops when confident.

Approach

GUI grounding is difficult because dense screenshots often contain many small, similar, or closely placed elements. A wrong click can break the task flow of a computer-use agent. DART addresses this by refining the prediction before execution, instead of relying on expensive post-action reflection.

Core Idea

DART treats attention density as a confidence score. Concentrated attention usually means the model is confident and close to the correct target. Dispersed attention suggests uncertainty, so the method keeps more visual context and refines the prediction with a cropped view.

Key Components

  • Confidence estimation: computes confidence from the density of the attention map.
  • Adaptive cropping: uses large crops for uncertain predictions and smaller crops for confident ones.
  • Early stopping: stops refinement when confidence is high enough.
  • Coordinate mapping: maps refined local predictions back to the original screenshot.
  • Model generalization: supports both attention-based and direct coordinate prediction models.

Evaluation

  • Benchmarks: ScreenSpot-Pro, UI-Vision, and OSWorld-G.
  • Models: GUI-Actor-7B, V2P-7B, Qwen3VL-8B, UI-Venus-1.5-8B, and KV-Ground-8B.
  • Metric: grounding accuracy, based on whether the predicted point falls inside the target bounding box.

Results

DART consistently improved GUI grounding accuracy across different models and benchmarks, with the strongest gains on dense, high-resolution interfaces.

Main Results

Dataset Model Baseline DART Gain
ScreenSpot-Pro GUI-Actor-7B 43.8% 60.5% +16.7%
ScreenSpot-Pro V2P-7B 49.5% 64.2% +14.7%
ScreenSpot-Pro KV-Ground-8B 73.0% 80.9% +7.9%
UI-Vision V2P-7B 23.9% 33.9% +10.0%
OSWorld-G GUI-Actor-7B 48.4% 61.4% +13.0%
OSWorld-G V2P-7B 52.0% 62.8% +10.8%

Key Takeaways

  • DART improved both attention-based and coordinate-prediction grounding models.
  • The largest gain was +16.7% on ScreenSpot-Pro with GUI-Actor-7B.
  • KV-Ground-8B with DART reached 80.9% accuracy on ScreenSpot-Pro.
  • Adaptive refinement worked better than fixed-scale zooming and other refinement baselines.
  • The method improves accuracy without additional training or model architecture changes.

Limitations

DART still depends on the initial prediction. If the first prediction is far from the target, refinement may crop the wrong region. It also adds extra inference cost, although early stopping helps avoid unnecessary refinement.
