Map LinkedIn Email

CADS - FPT Corporation · 2024.03

Privacy-aware workflow to enrich email-based customer records with LinkedIn profile information. The pipeline checks whether each email can be matched, then normalizes profile, work-experience, and education data for downstream analytics.

GitHub Go to Approach

Demo

Demo flow: email sample → authorized profile lookup → classify match result → parse profile response → normalize profile/work/education data → aggregate and evaluate data quality.

Email No Match

Email Match

  • Input sample size: 50,000 emails.
  • Matched to LinkedIn accounts: 4,070 (8.1%).
  • Profiles with work-experience data: 2,041 (4.1%).
  • Profiles with education data: 1,560 (3.1%).

Approach

  • Main scripts: 1_get_token_from_outlook.py and 2_email_in_linkedin.py.
  • Lookup outcomes are classified into matched profile, partial match, no match, and failed lookup.
  • Matched responses are parsed into structured profile, work-experience, and education tables.
  • Expected outputs: access_token.json and email_match_linkedin.json.
  • Local resources include resource/email.xlsx, resource/account.txt, and optional resource/chromedriver.

Key Findings

  • The workflow matched 4,070 / 50,000 emails to LinkedIn accounts.
  • 2,041 profiles included work-experience information.
  • 1,560 profiles included education information.
  • Education data was generally more complete than work-experience data.
  • The enriched attributes can support customer profiling, segmentation, and professional-background analysis when used with proper authorization and privacy controls.

Compliance Note

Use only with authorized accounts and approved data-processing permissions. Credentials, tokens, and personally identifiable data should remain in controlled local/private environments.
